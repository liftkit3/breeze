# Breeze M1 S1 — Post-mortem: 4 hours to start a blank app

**Date:** 2026-05-03
**Story:** M1 S1 — Monorepo + Expo + NativeWind locked
**Time lost:** ~4h

---

## What happened

Trying to start a blank Expo app in an npm workspaces monorepo with
NativeWind styling. Hit 6 sequential errors before Metro started clean.

---

## Error chain

| # | Error | Root cause |
|---|-------|------------|
| 1 | `Unable to resolve ../../App` | `"main": "expo-router/entry"` doesn't resolve reliably in monorepos before Metro's nodeModulesPaths applies. Falls back to `expo/AppEntry.js` which looks for a local `App.js`. |
| 2 | `Cannot find module 'react-native-worklets/plugin'` | `nativewind@4.2.3` (resolved from `^4.1.23`) requires Reanimated 4. We were on Reanimated 3.16. |
| 3 | `Unable to resolve react-native-css-interop/jsx-runtime` | `expo-router@4.x` + `react-native-css-interop@0.1.22` version mismatch. Both caused by wrong SDK version matrix. |
| 4 | `react@19.2.5` vs `react-native-renderer@19.1.0` | A transitive dep pulled React up one minor. react-native's renderer was still pinned to 19.1.0. App crashed on every render. |
| 5 | Stale versions after fixing package.json | Old lockfile kept deduplicating `react@18` to root. `npm install` on top of a broken lockfile patches, doesn't fix. |

---

## Root cause

**CLAUDE.md had a wrong version matrix.**
`expo ~54.0.0` was paired with `react@18.3`, `react-native@0.76`,
`reanimated ~3.16` — which are **SDK 52's** companions.
SDK 54 ships with React 19, RN 0.81, Reanimated 4.
Every single error was a downstream consequence of this one wrong assumption.

---

## Fixes applied

### 1. Correct SDK 54 version matrix

```json
"expo":                       "~54.0.0"
"react":                      "19.1.0"
"react-native":               "0.81.5"
"expo-router":                "~6.0.23"
"react-native-reanimated":    "~4.1.1"
"nativewind":                 "^4.2.1"
"tailwindcss":                "^3.4.17"
```

### 2. Pin React hard via root overrides

In root `package.json`:

```json
"overrides": {
  "react": "19.1.0",
  "react-dom": "19.1.0"
}
```

### 3. Concrete entry point

`apps/mobile/index.js`:

```js
import "expo-router/entry";
```

`package.json` → `"main": "index"`

### 4. Clean reinstall after version changes

```bash
rm -rf node_modules apps/mobile/node_modules package-lock.json
npm install
```

---

## Rules going forward

| Rule | Why |
|------|-----|
| Never write version matrices from memory — always check SDK release notes | Cost 4h here |
| Never use `^` on RN ecosystem deps (nativewind, reanimated, css-interop) | `^4.1.23` silently jumped to breaking 4.2.3 |
| Always add `overrides.react` in root `package.json` before first install | Transitive deps will drift React's version |
| When changing major versions: delete lockfile + all node_modules first | Stale lockfile hid wrong versions after fixes |
| Always use a concrete `index.js` as entry, never `"main": "expo-router/entry"` | Metro resolves entry before nodeModulesPaths applies |

---

## What the correct setup takes (once you know)

1. Read SDK release notes → write correct versions (~15 min)
2. Add `overrides.react` to root `package.json`
3. Create `index.js` entry point
4. `npm install` fresh
5. `expo start` — done

**Total: ~45 min, not 4 hours.**
